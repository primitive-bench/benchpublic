# golden-sets-public / extraction

PUBLIC dev examples for the **web_extraction** primitive. Reproducible by anyone.

**Held-out test answers NEVER live here** — they sit behind the private eval
server (primitivebench-platform/apps/eval-server) with HMAC-keyed split integrity
(D-07). The real public split is **generated by the registry-delta pump**
(`bench_core.goldgen`) from authoritative registries and is kept out of git; the
file here is hand-written illustration only.

## What extraction scores

web_extraction and web_search probe the **same golden rows**. Extraction hands a
vendor the row's `canonical_url` and asks: did `truth_token` survive into the
vendor's extracted main content? Scoring is deterministic whitespace-normalized
substring — no judge (see `eval-extraction/scoring.py`). A miss is decomposed into
`blocked` / `truncated` / `token_absent` / `empty` so an anti-bot wall is never
charged as an extraction failure ("the gap was anti-bot blocking, not freshness").

## GoldenRow schema (the frozen row contract)

| field | meaning |
|---|---|
| `row_id` | stable id; HMAC message for the split (salt-independent) |
| `query` | the descriptive query (also used by web_search) |
| `canonical_url` | the page handed to the extractor |
| `equivalence_members` | cross-domain mirrors that also count as correct |
| `truth_token` | the fact that must survive extraction (FR doc no. / CVE id / SEC accession / release tag) |
| `authoritative_timestamp` | ISO-8601 from the source of truth |
| `stratum` | difficulty kind: `navigational`, `long_tail`, `fresh`, `sentinel`, `sos` → maps to GroundTruthTier |
| `source` | registry of origin: `sec_edgar`, `fed_register`, `nvd_cve`, `github_releases`, `sentinel` |
| `split` | `public` (here) or `holdout` (private eval server only) |
| `verified_at` | the ≥2 verification fetch timestamps |
| `token_depth` | **offset of the token in canonical main content** — the extraction discriminator (title-zone ~0 vs deep-body) |
| `canonical_chars` | length of canonical main content |
| `slices` | intent tags (shared with web_search) |

`dev.example.jsonl` carries 4 illustrative rows chosen to show the extraction
story: a deep-body Federal Register token on an anti-bot site (`blocked` misses),
a title-zone CVE id (trivial survival floor), a very long SEC filing (`truncated`
risk), and a JS-rendered GitHub release.
